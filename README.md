# Overview

*opnsense-ingest* is a lightweight ingestion skill for OPNsense firewalls that takes structured JSON output from the **opnDossier** CLI and stores it as timestamped snapshots on disk. It is designed for multi-firewall environments where you regularly export config.xml, generate findings, and want a durable history of each run.

Each call creates a new snapshot file containing the full findings plus metadata such as firewall host, snapshot time, and source. This makes it easy to build follow-up skills for querying rules, dead rules, security issues, or configuration drift across snapshots and devices.

## What this skill does

- Accepts a JSON payload generated by opnDossier *convert -f json* for any OPNsense config.xml.
- Writes the findings to a persistent workspace directory, grouped by firewall host and timestamp.
- Adds useful metadata (host, snapshot time, source, ingestion timestamp).
- Returns a short summary (dead rules and security issues count) together with the file path.
- Automatically cleans up old snapshot files per host after 30 days, keeping the directory tidy.

### Typical use case:

1. A cron job or bash script copies ***config.xml*** from one or more OPNsense firewalls.
2. The script runs opnDossier to generate ***findings.json***.
3. The script calls the OpenClaw */tools/invoke* API with *tool: "opnsense-ingest"* and passes the findings.
4. OpenClaw stores the snapshot in its workspace and returns a status plus a quick summary.

#### Inputs
The skill expects the following arguments (mapped 1:1 to **inputs** in the frontmatter):

- *firewallHost* (string, required): Hostname or IP of the OPNsense firewall, e.g. *fw.node.org*.
- *snapshotTime* (string, required): Timestamp of the snapshot, typically YYYYMMDD-HHMMSS.
- *source* (string, required): Source of the analysis, usually "*opnDossier*".
- *findings* (object, required): The full JSON findings from opnDossier *convert -f json*, including sections like *dead_rules*, *security_issues*, and others.

Example payload sent to OpenClaw:

```
{
  "tool": "opnsense-ingest",
  "action": "json",
  "args": {
    "firewallHost": "fw.node.org",
    "snapshotTime": "20260313-030000",
    "source": "opnDossier",
    "findings": { "...": "full opnDossier JSON here" }
  }
}
```
#### Outputs
On success, the skill returns a JSON object like:

```
{
  "ok": true,
  "saved": "/home/openclaw/.openclaw/workspace/opnsense/snapshots/fw.node.org/20260313-030000-opnDossier.json",
  "host": "fw.node.org",
  "time": "20260313-030000",
  "summary": "2 dead rules, 1 security issues",
  "status": "ok",
  "totalFiles": "5"
}
```
Key fields:

- *saved:* Full path to the stored snapshot JSON file.
- *summary:* Short text summary derived from *findings.dead_rules* and *findings.security_issues*.
- *totalFiles:* Number of snapshot files currently present for that host.

On invalid input (missing required fields), it returns:

```
{
  "ok": false,
  "error": "Faltan firewallHost, snapshotTime o findings",
  "status": "error"
}

```
#### Storage layout
By default, snapshots are stored under the OpenClaw workspace:

```
$OPENCLAW_WORKSPACE/opnsense/snapshots/<firewallHost>/<snapshotTime>-<source>.json
# e.g.
~/.openclaw/workspace/opnsense/snapshots/fw.node.org/20260313-030000-opnDossier.json
```
This layout makes it straightforward to:

- Browse historical snapshots per firewall.
- Build additional skills that read and aggregate findings across hosts.
- Back up or mirror the OpenClaw workspace directory.

### Security and permissions
- Uses only the filesystem permission with read/write access limited to the OpenClaw workspace directory.
- Does not execute external commands and does not perform any network requests.
- All sensitive firewall configuration data stays on the same host where OpenClaw is running, inside the workspace directory.

You should still treat the workspace snapshots as sensitive configuration data and include them in your usual backup and access control strategy for OpenClaw.

### When to use this skill
Use opnsense-ingest whenever you:

- Periodically audit OPNsense firewalls with opnDossier and want a structured, queryable history.
- Need a simple ingestion layer that separates “collect findings” (cron/scripts) from “analyze findings” (other OpenClaw skills).
- Plan to build follow-up skills that answer questions like:
    - “Show WAN rules for host X in the latest snapshot.”
    - “List dead rules across all firewalls this week.”
    - “Highlight new security issues compared to yesterday’s snapshot.”
